High-Level Definition of Human-Robot Interaction Patterns

Polimi has developed a framework for developing robotic applications in service settings (such as hospitals, home assistance, education) that can handle the unpredictability of human behavior.
Analysis at design time (i.e., before the interactive scenario is deployed or simulated) entails a manual scenario configuration. The application designer defines the geometrical layout where the humans and the robot operate, the robots' characteristics, the humans' characteristics, and how the humans and the robot will interact. The latter always conforms to pre-defined interaction patterns such as "Human Follower" and "Human Leader," capturing the human following or leading the robot to a specific destination.
This project aims at developing a user-friendly notation to define custom interaction patterns should the pre-determined set fail to capture the desired contingency. The candidate will:
  1. identify and abstract the characteristics that define an interaction pattern and distinguish it from others in the set;
  2. develop a notation (from scratch or extending a pre-existing one) to define such characteristics in an accessible fashion;
  3. validate the contribution through examples of human-robot interactions inspired by realistic scenarios.

Automated Mission Re-Planning for Assistive Robotics Applications

Polimi has developed a framework for developing robotic applications in service settings (such as hospitals, home assistance, education) that can handle the unpredictability of human behavior. The framework includes a design-time analysis phase to estimate the most likely outcome of the mission (e.g., success vs. failure due to low battery charge) and, subsequently, deploy (or simulate) the application. Due to the uncertainties in the environment, the outcome observed at runtime may not match the one estimated offline, thus, requiring a reconfiguration of the mission before running new simulations.
The reconfiguration is currently performed manually by the framework's user. This thesis project aims to automate the reconfiguration (or "re-planning") phase by developing a tool to compute alternative plans for the user to choose from and automatically repeat the design-time verification.
The thesis outcome should include a solid justification for the chosen re-planning strategy and validation on significant use cases, highlighting the re-planning phase's added value.
